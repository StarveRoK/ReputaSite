## Соглашения по стилям

- Каждая секция подключает свой CSS-файл (`import '@/styles/block_N.css'`) напрямую в компонент.
- Кастомные классы используют префикс `cst-`.
- Глобальные CSS-переменные — только в `globals.css`.
- TailwindCSS — для утилитарных классов; Bootstrap — для сетки и компонентов.
- Всегда использовать CSS-переменные из `globals.css` вместо хардкоженных значений (`var(--color-primary)` вместо `#0163FC`).

### Медиа-запросы

Подход desktop-first (`max-width`). Порядок брейкпоинтов — от большего к меньшему:

```css
@media (max-width: 1280px) { }  /* Десктоп */
@media (max-width: 1024px) { }  /* Ноутбуки */
@media (max-width: 768px)  { }  /* Планшеты */
@media (max-width: 480px)  { }  /* Мобильные */
```

Обратный порядок приведёт к тому, что стили меньших брейкпоинтов будут перекрыты большими.

## Данные и Strapi

Все текстовые данные, ссылки и контент выносятся в константы вверху компонента — в будущем они будут заменены на fetch из Strapi API.

**Правила:**
- Никакого хардкода текста прямо в JSX — только через константу.
- Никакого `dangerouslySetInnerHTML` — вместо него использовать `rich()`.
- Форматирование (`<b>`, `<br/>`, цвет) хранится внутри строки-константы и парсится через `rich()`.
- Массивы однотипных элементов (бенефиты, изображения, ссылки) — всегда массив объектов, рендер через `.map()`.

## Утилита rich()

`src/lib/richText.tsx` экспортирует `rich(text, tags?, markers?)` — безопасная альтернатива `dangerouslySetInnerHTML`. Парсит строку и возвращает `ReactNode[]`.

**Встроенные теги:** `<b>`, `<i>`, `<strong>`, `<br/>`

```tsx
import { rich } from '@/lib/richText';

// Встроенные теги
rich('Попробовать <b>бесплатно</b> — 14 дней')

// Кастомный тег
rich('Цена <accent>990₽</accent> в месяц', {
    accent: (c, k) => <span key={k} className={s.accent}>{c}</span>,
})

// Маркер-символ (% оборачивает часть текста)
rich('Управляем %репутацией% для вас', undefined, {
    '%': (c, k) => <span key={k} className={s.blue}>{c}</span>,
})
```

Можно сочетать все три варианта в одной строке. Маркером может быть любой символ (`%`, `~`, `@`, `|`).

**BlueButton** принимает проп `label: string` — строка рендерится через `rich()`. Старый API `children` сохранён для обратной совместимости с блоками, которые ещё не мигрированы.

```tsx
// Новый способ — через label
<BlueButton href="..." label="Попробовать <b>бесплатно</b>" />

// Старый способ — через children (пока поддерживается)
<BlueButton href="...">Попробовать бесплатно</BlueButton>
```

## API-слой

`src/lib/api.ts` экспортирует `apiFetch<T>(endpoint, options?)`:
- Базовый URL: `process.env.NEXT_PUBLIC_API_URL` (дефолт: `http://localhost:8000/`)
- Добавляет `Content-Type: application/json`
- ISR-ревалидация: 60 секунд (`next: { revalidate: 60 }`)
